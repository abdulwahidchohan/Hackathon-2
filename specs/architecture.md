# System Architecture — Phase I

## Scope

Phase I is a single-process, in-memory console application. No network, no database, no persistence.

## Components

1. **CLI entrypoint** — Parses command-line arguments and delegates to the todo service. Entry: `todo.cli:main`.
2. **Todo service / store** — In-memory list of tasks. Exposes: add, list, get, update, delete, complete.
3. **Task model** — Immutable or simple dataclass/dict: id, title, description, completed, created_at (and optionally updated_at).

## Data flow

```
User → CLI (argparse or similar) → Todo store (in-memory list) → Response to user
```

- No disk I/O. No external services.
- IDs: Generated by the store (e.g. incremental integer). Unique within a run.

## Interfaces

- **CLI commands:** `add`, `list`, `update`, `delete`, `complete`.
- **Store API (internal):** add_task(title, description?), list_tasks(), get_task(id), update_task(id, title?, description?), delete_task(id), complete_task(id).

## Technology

- Python 3.13+, stdlib only for Phase I (or one CLI library if specified in constitution). UV for dependency and script execution.

## Out of scope (Phase I)

- REST API, web UI, database, authentication, multi-user, file persistence.
